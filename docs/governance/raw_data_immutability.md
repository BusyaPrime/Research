# Политика raw data immutability

## Инвариант

Raw layer не редактируется задним числом. Что приехало от источника, то и хранится как исходный факт, даже если дальше мы нашли проблему, нормализовали схему или почистили запись в bronze/silver.

## Зачем это нужно

- для provenance;
- для разбирательства QA-инцидентов;
- для повторной нормализации без потери исходника;
- для честного diff между поставками данных.

## Разрешенные операции

- добавить request manifest;
- сохранить raw payload;
- пометить batch метаданными;
- положить рядом failed extract или QA diagnostics.

## Неразрешенные операции

- переписать raw payload “как должно было быть”;
- стирать проблемные строки молча;
- смешивать raw и normalized слой в одной таблице.
