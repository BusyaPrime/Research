# Глоссарий

## Основные термины

**Decision timestamp**  
Момент, в который стратегия имеет право принять решение. В этой системе он расположен после `close_t`.

**Execution timestamp**  
Момент исполнения решения. Для baseline path это `open_{t+1}`.

**Point-in-time / PIT**  
Режим, в котором каждая запись, join и feature используют только те данные, которые реально были доступны на момент решения.

**OOF prediction**  
Прогноз, полученный на наблюдениях, которые не использовались для fit модели в соответствующем фолде.

**Universe snapshot**  
Point-in-time снимок eligible securities на конкретную дату с причинами включения/исключения.

**Gold panel**  
Готовая к моделированию панель, где собраны features, labels, row diagnostics и metadata.

**Purge / embargo**  
Механизмы, которыми убирается overlap и утечки между train/valid/test на временной оси.

**Gross / net**  
Gross — результат без учета торговых затрат. Net — результат после costs и borrow assumptions.

**Capacity**  
Оценка того, как стратегия ведет себя на разных уровнях AUM при ограничениях на участие в обороте и impact.

**Temporary simplification**  
Честно помеченное упрощение относительно полного spec, с описанием причины, влияния и пути доработки.
