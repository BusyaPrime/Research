# План демонстрации результата

## Цель демонстрации

Показать не просто “модель дала цифры”, а что собрана системная research platform, где результат держится не на магии ноутбука, а на воспроизводимом пайплайне и честной временной дисциплине.

## Порядок показа

1. Коротко показать source of truth и архитектурные инварианты.
2. Показать data lineage: manifests, config hash, dataset version, run id.
3. Показать PIT/universe/label semantics, потому что это сердцевина честности.
4. Показать OOF path и walk-forward.
5. Показать portfolio/backtest: gross, net, turnover, costs.
6. Показать capacity/regime/decay.
7. Показать ограничения и временные упрощения без попытки их прятать.

## Что нужно иметь под рукой

- final report;
- review bundle;
- release checklist;
- implementation notes;
- 2-3 точечных примера leakage guards и PIT tests.

## Чего не делать на демонстрации

- не уходить в обсуждение только модельного score;
- не пытаться замазать временные упрощения;
- не показывать только лучший график без артефактов вокруг.
